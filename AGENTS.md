# noia-ui

TypeScript UI component library built with Vite library mode.

## Commands

- `pnpm dev` — dev server (Vite)
- `pnpm build` — `tsc && vite build` (typecheck must pass before build)
- `pnpm preview` — preview production build

No test, lint, or format commands are configured yet.

## Build & Output

- **Entry**: `src/index.ts`
- **Formats**: ES (`dist/index.es.js`) and UMD (`dist/index.umd.js`)
- **Global name**: `NoiaUI`
- **Types**: generated by `vite-plugin-dts` → `dist/index.d.ts`
- **CSS**: consumers import via `noia-ui/style.css` (subpath export in package.json)
- `tsconfig.json` has `noEmit: true` — `tsc` is type-check only; Vite handles compilation

## TypeScript Constraints

- **`verbatimModuleSyntax`** — use `import type` for type-only imports
- **`erasableSyntaxOnly`** — no enums, no parameter properties, no namespace merging
- **`strict: true`** — full strict mode
- **Path alias**: `@` → `src` (configured in both tsconfig and vite)

## Architecture

- Each component lives in `src/<component>/index.ts` and is re-exported from `src/index.ts`
- Styles go in `src/styles/` as `.scss` files (Sass is a dev dependency)
- `index.html` is a dev playground, not published
